# SEC-001 — Required sections present

> Category **SEC** (sections) · Scope **document** · Default severity **error** · Fixable **yes**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Every Markdown file in scope must contain a set of required **sections**, identified by heading
text. SEC-001 collects the file's section headings and reports a finding for each required section
name that does not appear.

This keeps documents anchored to an agreed outline — e.g. every ADR must carry a `Context`, a
`Decision`, and a `Consequences` section. SEC-001 only checks that each section *exists*; use
[SEC-002](SEC-002.md) to constrain the order those sections appear in, and
[SEC-003](SEC-003.md) to check a whole family of files against a reference template.

Matching is by exact section heading text, at any heading depth. A missing section has no line to
anchor to, so its finding is reported at line 0. If `files`/`exclude` are set, only matching files
are checked; otherwise every included file is checked.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `sections` | `string[]` (≥1) | **yes** | — | Section heading names every in-scope file must contain. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "SEC-001", "options": { "sections": ["Intro", "Summary"] } }
  ]
}
```

### Passes

```md
# Title

## Intro

## Summary
```

### Fails

```md
# Title

## Intro
```

→ SEC-001 reports that the required section **`Summary`** is missing, at line 0 of the file.

### Scoped to a document family

```jsonc
{ "rule": "SEC-001", "options": { "sections": ["Context", "Decision"], "files": ["adr/**/*.md"] } }
```

Only files under `adr/` are required to carry both sections; documents elsewhere are ignored.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- **Fixable:** running with `--fix` appends a scaffold for each missing section at the end of the
  file — a `## <Section>` heading followed by a `TODO` body. For the example above the fix would
  append:

  ```md
  ## Summary

  TODO
  ```

  The fix always writes level-2 (`##`) headings and never removes or reorders existing content;
  fill in the `TODO` body afterwards.
- **`files`/`exclude`** let you register SEC-001 multiple times with different required outlines
  for different document families.
- Section comparison is on exact heading text; it does not normalize case or punctuation.

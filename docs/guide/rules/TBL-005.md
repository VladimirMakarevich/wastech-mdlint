# TBL-005 — Cross-column conditional

> Category **TBL** (tables) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Within a single row, when one column satisfies a condition, another column must satisfy its own
condition. TBL-005 evaluates the `when` condition against each row; when it holds, the row's
`then` condition must also hold, and a finding is reported otherwise.

This expresses dependencies the single-column rules cannot — "if `Status` is `done`, then
`Resolution` must be non-empty", or "if `Type` is `bug`, then `Severity` must match `^P[0-3]$`".
The `when` clause selects the rows that are subject to the requirement; the `then` clause is the
requirement.

Each condition targets one `column` and carries **exactly one** of `equals`, `matches`, or
`notEmpty`. Rows in a table that does not declare *both* the `when` and `then` columns are
skipped (there is nothing to assert against). If `section` is set, only tables under a heading
whose text matches that section are checked.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `when` | condition object | **yes** | — | The trigger: which rows the requirement applies to. |
| `then` | condition object | **yes** | — | The requirement that must hold on triggered rows. |
| `section` | `string` | no | — | Only check tables under a heading matching this text; omit to check all tables. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

A condition object has these fields:

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `column` | `string` | **yes** | Header name the condition reads. |
| `equals` | `string` | one of these three | Passes when the trimmed cell equals this exact string. |
| `matches` | `string` | one of these three | Passes when the trimmed cell matches this regex source. |
| `notEmpty` | `boolean` | one of these three | When `true`, passes when the trimmed cell is non-empty. |

## Example

### Config

```jsonc
{
  "rules": [
    {
      "rule": "TBL-005",
      "options": {
        "when": { "column": "Status", "equals": "done" },
        "then": { "column": "Resolution", "notEmpty": true }
      }
    }
  ]
}
```

### Passes

```md
| Status | Resolution |
| ------ | ---------- |
| open   |            |
| done   | fixed      |
```

The `open` row is not triggered (its `Status` is not `done`), so its empty `Resolution` is fine.
The `done` row is triggered and its `Resolution` is non-empty.

### Fails

```md
| Status | Resolution |
| ------ | ---------- |
| done   |            |
```

→ TBL-005 reports that when **`Status`** is `done`, **`Resolution`** must satisfy its condition
(here `notEmpty`) but the cell is empty, anchored at that row.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- Both conditions compare against the **trimmed** cell value.
- A row is only checked when its table declares both columns; use [TBL-001](TBL-001.md) if the
  columns themselves must be present.
- The condition schema requires one of `equals`/`matches`/`notEmpty`; a condition with none is a
  config error.

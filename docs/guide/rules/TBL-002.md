# TBL-002 — Non-empty table cells

> Category **TBL** (tables) · Scope **document** · Default severity **warning** · Fixable **yes**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Cells in the target column(s) of every in-scope table must not be empty. TBL-002 reads each
table row and reports a finding for any cell whose value is blank after trimming whitespace.

This catches half-filled tables — a requirements row with no `Owner`, a status table with a
missing `Status`. TBL-001 checks that a column *exists*; TBL-002 checks that its cells are
actually *populated*. Use [TBL-003](TBL-003.md)/[TBL-004](TBL-004.md) to further constrain what
those non-empty values may be.

If `columns` is omitted, every column of each in-scope table must be non-empty. If `section` is
set, only tables under a heading whose text matches that section are checked; otherwise every
table in the file is checked.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `columns` | `string[]` (≥1) | no | all columns | Header names whose cells must be non-empty; omit to require every column. |
| `section` | `string` | no | — | Only check tables under a heading matching this text; omit to check all tables. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "TBL-002", "options": { "columns": ["Owner"] } }
  ]
}
```

### Passes

```md
| ID    | Owner | Status |
| ----- | ----- | ------ |
| REQ-1 | Ann   | open   |
```

### Fails

```md
| ID    | Owner | Status |
| ----- | ----- | ------ |
| REQ-1 | Ann   | open   |
| REQ-2 |       | bogus  |
```

→ TBL-002 reports that the cell in column **`Owner`** on the `REQ-2` row is empty, anchored at
that row's line. Because only `Owner` is scoped, the empty-looking `Status` value `bogus` is not
this rule's concern.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- **Fixable:** running the linter with `--fix` replaces each empty target cell with ` TODO ` so
  the table stays well-formed and the placeholder is easy to find and fill in. The fix is applied
  in place; on re-lint the finding clears (the cell now has content). Non-canonical rows (no
  leading pipe) or cells beyond the row's pipe count are skipped rather than risk corrupting the
  source.
- **Emptiness** is checked after trimming: a cell containing only spaces counts as empty.
- **`files`/`exclude`** let you register the rule multiple times with different column sets for
  different document families.

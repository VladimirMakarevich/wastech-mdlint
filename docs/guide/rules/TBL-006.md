# TBL-006 — Unique column IDs across files

> Category **TBL** (tables) · Scope **project** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Values in the target column must be unique across every in-scope table in the whole repository,
not just within one file. TBL-006 scans the corpus in deterministic path order and reports a
finding on the **second and later** occurrence of any repeated value, naming where the value was
first seen.

This keeps identifier spaces collision-free — a `REQ-1` that appears in two requirement files, a
duplicated ticket `ID`, a reused anchor key. Unlike every other TBL rule, this one is
project-scoped: it reasons about all documents at once.

Empty cells are ignored. When `idPattern` is set, only cells matching that pattern are treated as
IDs and considered for uniqueness; non-matching cells are skipped, which lets a column hold a mix
of IDs and free text. If `section` is set, only tables under a heading whose text matches that
section participate.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `column` | `string` | **yes** | — | Header name of the column whose values must be unique. |
| `idPattern` | `string` | no | — | Regex source; only cells matching it are treated as IDs and checked. |
| `section` | `string` | no | — | Only include tables under a heading matching this text; omit to include all tables. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "TBL-006", "options": { "column": "ID" } }
  ]
}
```

### Passes

`a.md`

```md
| ID    |
| ----- |
| REQ-1 |
```

`b.md`

```md
| ID    |
| ----- |
| REQ-2 |
```

### Fails

`a.md`

```md
| ID    |
| ----- |
| REQ-1 |
```

`b.md`

```md
| ID    |
| ----- |
| REQ-1 |
```

→ TBL-006 reports the duplicate **`REQ-1`** in `b.md`, noting it was first seen in `a.md`. The
finding is attributed to the later occurrence; the first occurrence is left clean and surfaced as
`firstSeenIn` for actionable diagnostics.

### Restricting to real IDs

```jsonc
{ "rule": "TBL-006", "options": { "column": "Key", "idPattern": "^REQ-\\d+$" } }
```

Only cells matching `^REQ-\d+$` are checked for uniqueness; a `Key` cell holding a note like
`n/a` is ignored.

## Notes

- **Scope:** `project` — evaluated across the whole corpus, not per file. This is the only
  project-scoped TBL rule.
- The corpus is scanned in sorted repository-relative path order, so which occurrence counts as
  "first" is deterministic and stable across runs and platforms.
- Values are **trimmed** before comparison; empty cells never participate.
- **`files`/`exclude`** narrow the participating file set — useful to scope the uniqueness domain
  to one document family (e.g. `docs/requirements/**`).

# REF-001 — Relative links resolve to a file

> Category **REF** (references) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Every relative Markdown link in scope must point at a file that actually exists. REF-001 reads
each local-file link, resolves its target, and reports a finding when nothing is there.

A link target "resolves" when it is either part of the analyzed Markdown corpus or exists on
disk under the repository root. The on-disk check means links to non-Markdown files (assets,
scripts, files outside `include`) still pass as long as the file is present.

Two target shapes are handled:

- **Relative targets** (`../design/spec.md`, `./notes.md`) resolve against the linking file's
  directory.
- **Root-relative targets** (`/guides/setup`) resolve through the [site router](#site-router)
  when one is configured, mapping the URL to candidate source files; without a router they are
  treated as repository-root-relative paths.

Only the file part of a target is checked here. Anchors (`spec.md#section`) are validated by
[REF-002](REF-002.md), and same-file anchors and non-local schemes (`https:`, `mailto:`) are out
of scope. This is a **local-only** check: REF-001 never performs network or HTTP link checking.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `exclude` | `string[]` | no | — | Glob(s) matching link *targets* to skip (e.g. generated paths that never exist on disk). |
| `siteRouter` | `object` | no | inherits `settings.siteRouter` | Overrides the site router used to resolve root-relative links for this rule instance. See [Site router](#site-router). |

The `siteRouter` object accepts `preset`, `contentDir`, and `defaultLocale` (all optional
strings).

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "REF-001" }
  ]
}
```

### Passes

```md
See the [design spec](../design/spec.md) and the [logo](./assets/logo.png).
```

→ Both targets exist on disk, so REF-001 reports nothing.

### Fails

```md
See the [design spec](../design/spce.md).
```

→ REF-001 reports that link target **`../design/spce.md` does not resolve to a file**, anchored
at the link.

### Excluding a target pattern

```jsonc
{ "rule": "REF-001", "options": { "exclude": ["generated/**"] } }
```

Links whose target matches `generated/**` are skipped, so a link to a path produced by a later
build step is not flagged.

## Site router

Root-relative links such as `/guides/setup` do not name a file directly; a static-site generator
maps them to source files. `settings.siteRouter` describes that mapping and is **inherited by
reference rules**; a per-rule `siteRouter` option overrides the inherited value for that instance.

```jsonc
{
  "settings": {
    "siteRouter": { "preset": "starlight", "contentDir": "src/content/docs", "defaultLocale": "en" }
  }
}
```

With the `starlight` preset, `/guides/setup` resolves against candidates such as
`src/content/docs/guides/setup.md` and `src/content/docs/guides/setup/index.md`. Links from a
non-default locale resolve same-locale first, then fall back to the default locale. Without a
router, a root-relative target is treated as a plain repository-root-relative path.

## Notes

- **Scope:** `document` — evaluated per file, but resolution consults the full corpus and the
  filesystem so cross-file targets are validated.
- **Local only:** resolution is against the corpus and the local filesystem. There is no external
  HTTP link checking and no external link cache in v2.
- Targets that escape the repository root never resolve.

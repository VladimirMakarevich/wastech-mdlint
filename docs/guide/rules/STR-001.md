# STR-001 — Required files exist

> Category **STR** (structure) · Scope **project** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

The project must contain every path listed in `files`. STR-001 scans the analyzed file corpus and
reports a finding for each required entry that nothing in the project satisfies.

This enforces repository structure — every project must ship a `README.md`, a `CONTRIBUTING.md`, a
`LICENSE`, and so on. Each entry may be a literal path or a glob, so you can require *that a
category of file exists* (e.g. at least one `docs/adr/*.md`) rather than one exact filename.

> **Note:** here `files` is the *required set* — the files that must be present — not a scoping
> filter. Unlike the document-scope SEC rules, STR-001 has no `files`/`exclude` scoping options;
> it always evaluates the whole project.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `files` | `string[]` (≥1) | **yes** | — | Paths or globs the project must contain; each entry unmatched by any file is reported. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "STR-001", "options": { "files": ["README.md", "CONTRIBUTING.md"] } }
  ]
}
```

### Passes

A project containing both `README.md` and `CONTRIBUTING.md` produces no findings.

### Fails

A project containing only `README.md`:

```md
# Readme
```

→ STR-001 reports that required file **`CONTRIBUTING.md`** is missing from the project, anchored at
that path.

### Requiring a category of file with a glob

```jsonc
{ "rule": "STR-001", "options": { "files": ["docs/adr/*.md"] } }
```

Passes as long as at least one file matches `docs/adr/*.md`; reports the glob as missing if none do.

## Notes

- **Scope:** `project` — evaluated once against the whole file corpus, not per document.
- **Glob entries** are satisfied when any project file matches; literal paths must match exactly.
- **Not fixable:** STR-001 cannot know the contents a missing file should have, so it reports rather
  than creating placeholder files.
- Each missing required entry is reported against its own path so results stay deterministic and
  point at the specific gap.

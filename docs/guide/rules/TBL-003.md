## What it checks

Every cell in the target column must hold one of an explicit set of allowed values. TBL-003 reads
each table row and reports a finding when a cell's value (after trimming) is not a member of the
`values` set.

This enforces controlled vocabularies â€” a `Status` column limited to `open`/`in-progress`/`done`,
a `Priority` column limited to `P0`/`P1`/`P2`. It complements [TBL-002](TBL-002.md) (which only
requires cells be non-empty) and [TBL-004](TBL-004.md) (which matches a pattern instead of an
enumerated set).

By default comparison is **case-sensitive**; set `caseSensitive: false` to compare
case-insensitively. If `section` is set, only tables under a heading whose text matches that
section are checked. Tables that do not declare the column are skipped.

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "TBL-003", "options": { "column": "Status", "values": ["open", "done"] } }
  ]
}
```

### Passes

```md
| ID    | Status |
| ----- | ------ |
| REQ-1 | open   |
| REQ-2 | done   |
```

### Fails

```md
| ID    | Status |
| ----- | ------ |
| REQ-1 | open   |
| REQ-2 | bogus  |
```

â†’ TBL-003 reports that the value **`bogus`** in column **`Status`** is not one of the allowed
values `open, done`, anchored at the `REQ-2` row.

### Case-insensitive comparison

```jsonc
{ "rule": "TBL-003", "options": { "column": "Status", "values": ["open", "done"], "caseSensitive": false } }
```

With `caseSensitive: false`, a cell holding `Done` or `OPEN` passes; with the default it would be
reported.

## Notes

- **Scope:** `document` â€” evaluated per file, independent of other files.
- Cell values are **trimmed** before comparison, so surrounding whitespace never causes a false
  positive.
- An empty cell is compared like any other value; it passes only if the empty string is in
  `values`. Pair with [TBL-002](TBL-002.md) if empty cells should also be flagged.

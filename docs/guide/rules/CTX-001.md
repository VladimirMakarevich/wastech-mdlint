## What it checks

Every section (the body of text under a heading) in scope must contain real content. CTX-001
reports a finding when a section's body is **empty** or consists of nothing but a single
**placeholder token**.

This catches half-drafted documents: a heading left with no prose beneath it, or a stub whose
only content is `TODO`. The classification is done on the *whole* trimmed body, not by substring â€”
so a paragraph that merely mentions a token (e.g. "we still need to fix the `TODO` list") is real
content and is **not** flagged.

Matching is case-insensitive, and a single trailing colon is ignored, so `TBD`, `tbd`, and `TBD:`
are all treated as placeholders. If `section` is set, only sections under a heading whose text
matches it are checked; otherwise every section in the file is checked.

### Placeholder tokens

The locked default placeholder set is:

`TBD`, `TODO`, `WIP`, `FIXME`, `N/A`

The `placeholders` option **extends** this set (union) â€” it never replaces it. The defaults always
remain active.

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "CTX-001" }
  ]
}
```

### Passes

```md
## Overview

This service ingests events and writes a normalized digest.
```

### Fails

```md
## Overview

TODO
```

â†’ CTX-001 reports that the section **`Overview`** contains only a placeholder, anchored at the
heading line. An empty section (a heading with no body) is instead reported as being empty.

### Extending the placeholder set

```jsonc
{ "rule": "CTX-001", "options": { "placeholders": ["LATER", "???"] } }
```

`LATER` and `???` now count as placeholders **in addition to** `TBD`, `TODO`, `WIP`, `FIXME`, and
`N/A`. A section whose entire body is `LATER` is flagged.

### Scoped to one section

```jsonc
{ "rule": "CTX-001", "options": { "section": "Acceptance criteria" } }
```

Only sections beneath an `Acceptance criteria` heading are checked; every other section is ignored.

## Notes

- **Scope:** `document` â€” evaluated per file, independent of other files.
- Detection is whole-body, not substring: prose that *mentions* a placeholder token is fine.
- Matching is case-insensitive and tolerates a single trailing `:`.
- **`files`/`exclude`** let you register the rule multiple times with different placeholder sets
  for different document families.

# LLM-001 — Eager-import context budget

> Category **LLM** (LLM context) · Scope **project** · Default severity **warning** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

For each configured **entrypoint** (e.g. an agent `CLAUDE.md`), LLM-001 follows its eager
`@path` imports transitively, sums the estimated token count of the entrypoint plus every file it
eagerly pulls in, and reports a finding when that total exceeds the per-entrypoint budget.

An eager import is the `@target` syntax that inlines another file's content. Because those files
become part of the entrypoint's context, their tokens count against the same budget. LLM-001
answers the question "how large is the fully-expanded context this entrypoint hands to a model?"
and flags the ones that have grown too big.

While traversing, LLM-001 also surfaces two structural problems it encounters: eager imports that
resolve to a file **not present** in the analyzed set, and eager-import **cycles**.

Import targets are resolved through the same shared resolver used by the ContextGraph builder and
the `REF` rules (honoring `settings.siteRouter` for root-relative targets), so LLM-001's view of
"what an entrypoint imports" matches the graph and compile budget rather than a separate ad hoc
resolver.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `entrypoints` | `string[]` (≥1) | **yes** | — | Glob(s) selecting the files treated as entrypoints; each matching file is budgeted separately. |
| `maxTokensPerEntrypoint` | `integer` (>0) | **yes** | — | Token budget for one entrypoint's own content plus its eager-import closure. |

## Example

### Config

```jsonc
{
  "rules": [
    {
      "rule": "LLM-001",
      "options": {
        "entrypoints": ["CLAUDE.md", "**/AGENTS.md"],
        "maxTokensPerEntrypoint": 8000
      }
    }
  ]
}
```

### Passes

```md
<!-- CLAUDE.md -->
# Project rules

Keep changes small. @docs/style.md
```

→ As long as `CLAUDE.md` plus the resolved `docs/style.md` estimate under `8000` tokens, LLM-001
reports nothing.

### Fails

```md
<!-- CLAUDE.md -->
# Project rules

Preamble @docs/big.md
```

→ With `docs/big.md` large enough that the entrypoint plus its import closure exceeds the budget,
LLM-001 reports at line 0 of `CLAUDE.md`: the entrypoint **is over context budget**, naming the
estimated total, the configured `maxTokensPerEntrypoint`, and the percentage over.

### Missing eager import

```md
<!-- CLAUDE.md -->
See @docs/missing.md
```

→ If `docs/missing.md` does not resolve to a known file, LLM-001 reports a **missing eager import**
at the import's line, naming both the raw `@docs/missing.md` target and the path it resolved to.

### Eager-import cycle

```md
<!-- a.md -->  @b.md
<!-- b.md -->  @a.md
```

→ When an entrypoint's closure contains a loop, LLM-001 reports an **eager import cycle**, listing
the cycle path (`a.md -> b.md -> a.md`). The cycle is broken during traversal, so it is not counted
repeatedly.

## Notes

- **Scope:** `project` — LLM-001 runs once over the whole analyzed document set (not per file). It
  reads the imports collected during the single parse pass rather than re-parsing.
- **Budget covers the eager closure.** Only eager `@path` imports are followed. Each imported file
  is counted once per entrypoint even if it is imported from several places.
- **Entrypoints are processed deterministically** in sorted path order, and each is budgeted on its
  own against `maxTokensPerEntrypoint`.
- **A single total budget.** LLM-001 budgets total tokens per entrypoint; there are no separate
  per-file-type sub-limits.
- **Token estimation is isolated and replaceable.** The count comes from one heuristic —
  `ceil(characters / 4)` — kept behind a single function so it can be swapped for a real tokenizer
  later without touching this rule. Treat the budget as an approximate guardrail, not an exact
  token count.

# REF-003 — Image targets resolve to a file

> Category **REF** (references) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Every relative image target in scope must point at a file that exists on disk. REF-003 reads each
image reference, resolves its target under the repository root, and reports a finding when the
file is missing.

Because image assets are usually not part of the analyzed Markdown corpus, resolution leans on a
filesystem existence check rather than the corpus. Relative targets (`./diagram.png`) resolve
against the image's own file; root-relative targets (`/img/logo.svg`) resolve against the
repository root.

Scheme-qualified targets are skipped: any target beginning with a scheme (`http:`, `https:`,
`data:`) is treated as external and never checked. Empty targets are ignored, and targets that
escape the repository root never resolve. This is a **local-only** check — REF-003 performs no
network requests and never fetches remote images.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `exclude` | `string[]` | no | — | Glob(s) matching image *targets* to skip (e.g. paths produced by a later build step). |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "REF-003" }
  ]
}
```

### Passes

```md
![architecture](./diagrams/architecture.png)
![hosted badge](https://example.test/badge.svg)
```

→ The local file exists and the external image is skipped, so REF-003 reports nothing.

### Fails

```md
![architecture](./diagrams/architecture.png)
![missing](missing.png)
```

with only `diagrams/architecture.png` present on disk.

→ REF-003 reports that image target **`missing.png` does not resolve to a file**, anchored at the
image's line.

### Excluding a target pattern

```jsonc
{ "rule": "REF-003", "options": { "exclude": ["build/**"] } }
```

Image targets matching `build/**` are skipped, so a reference to an asset generated later is not
flagged.

## Notes

- **Scope:** `document` — evaluated per file against the local filesystem.
- **Local only:** resolution is a filesystem existence check; there is no HTTP image checking and
  no remote fetch.
- External images are recognized by their URL scheme and are always skipped, so hosted badges and
  data URIs do not produce findings.

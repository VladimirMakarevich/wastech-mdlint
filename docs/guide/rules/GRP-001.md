# GRP-001 — No circular references

> Category **GRP** (graph integrity) · Scope **project** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Documents in the repository must not form a **dependency cycle**. GRP-001 reads the shared
[ContextGraph](../context-graph.md) and reports a finding for each cycle the graph reports — a
chain of documents where following references leads back to where it started
(e.g. `a.md → b.md → c.md → a.md`).

Cycles are computed once, corpus-wide, from the graph's edges. Every kind of reference edge
counts: plain Markdown links, in-page anchor links, image references, and eager `@import`
directives. When [`settings.idRef`](../configuration.md) is configured, `id-ref` edges (a
document that mentions an ID defined in another document) also participate, so an ID reference
can close a cycle just like a link does.

Because this rule reasons about relationships *between* files, its scope is `project`: it runs
once over the whole corpus rather than per file. The finding is attributed to the first document
in the cycle, anchored at the line of the first arc (the edge leaving that document); if the arc
has no known line, it anchors at the top of the file.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `siteRouter` | `{ preset?, contentDir?, defaultLocale? }` | no | — | Site-router hints accepted for forward-compatibility with route-aware analysis. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

> **Note:** the shared ContextGraph is built once for the whole corpus, so `files`, `exclude`, and
> `siteRouter` are accepted for forward-compatibility but do **not** re-scope which documents the
> cycle detection sees. Every document still participates in the graph.

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "GRP-001" }
  ]
}
```

### Passes

```md
<!-- index.md -->
See [details](detail.md).

<!-- detail.md -->
# Detail
```

`index.md` links to `detail.md`, and `detail.md` links nowhere back — acyclic.

### Fails

```md
<!-- a.md -->
See [b](b.md).

<!-- b.md -->
See [c](c.md).

<!-- c.md -->
See [a](a.md).
```

→ GRP-001 reports **`Dependency cycle detected: a.md -> b.md -> c.md -> a.md.`**, anchored at the
link in `a.md`.

### Cycle closed by ID references

With `settings.idRef` enabled, prose that mentions another document's ID forms an `id-ref` edge:

```jsonc
{
  "settings": {
    "idRef": { "idPattern": "^REQ-\\d+$", "definitions": ["*.md"], "idColumn": "ID" }
  },
  "rules": [{ "rule": "GRP-001" }]
}
```

```md
<!-- a.md -->
| ID |
| --- |
| REQ-1 |

See REQ-2 for context.

<!-- b.md -->
| ID |
| --- |
| REQ-2 |

See REQ-1 for context.
```

→ GRP-001 reports the cycle **`a.md -> b.md -> a.md.`** — the two ID references close the loop.
Without `settings.idRef`, no `id-ref` edges are built and there is no cycle to report.

## Notes

- **Scope:** `project` — evaluated once over the whole corpus, not per file.
- **Edges considered:** links, anchors, images, `@import`, and `id-ref` (the last only when
  `settings.idRef` is set). See [Context graph](../context-graph.md).
- **Attribution:** one finding per cycle, on the first document in the cycle at the first arc's
  line.

# SEC-003 — Sections conform to a template

> Category **SEC** (sections) · Scope **project** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Files in scope must carry every section heading that a reference **template** file declares.
SEC-003 reads the template named by `template`, collects its heading texts, and reports — for each
in-scope file — every template heading that the file is missing.

This checks a whole family of documents against one canonical outline: point `template` at a
`template.md` and every ADR, RFC, or runbook must reproduce its section headings. Unlike
[SEC-001](SEC-001.md), where you list the required sections inline, SEC-003 derives them from a real
file so the outline lives in one editable place.

Order is **not** enforced — only presence of each template heading; use [SEC-002](SEC-002.md) for
order. The template file itself is never checked against its own headings. Set `level` to compare
only headings at one depth (e.g. `2` compares only `##` headings). A file's own missing section is
reported at line 0 of that file.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `template` | `string` | **yes** | — | Repo-relative path to the reference file whose headings define the outline. |
| `level` | `integer` (≥1) | no | all depths | Only require template headings at this depth (e.g. `2` = `##`). |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files are checked against the template. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "SEC-003", "options": { "template": "template.md", "files": ["adr/**/*.md"], "level": 2 } }
  ]
}
```

### Template (`template.md`)

```md
# T

## Context

## Decision
```

### Passes (`adr/two.md`)

```md
# Two

## Context

## Decision
```

### Fails (`adr/one.md`)

```md
# One

## Context
```

→ SEC-003 reports that `adr/one.md` is missing section **`Decision`**, which is required by
template `template.md`.

## Notes

- **Scope:** `project` — the template is loaded once and every in-scope file is compared against it.
- **Missing template:** if `template` cannot be read, SEC-003 emits a single config-attributed
  error (`… was not found; skipping conformance checks.`) and performs no per-file checks, so a
  mistyped path never produces a flood of false positives.
- **Not fixable:** template conformance may need bodies as well as headings, so SEC-003 reports
  rather than rewrites. For an inline required-section list with a `--fix` scaffold, use
  [SEC-001](SEC-001.md).
- With `level` set, only template headings at that exact depth are required; a file need not
  reproduce the template's `# Title`.

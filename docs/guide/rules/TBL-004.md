# TBL-004 — Cell values match a pattern

> Category **TBL** (tables) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Every cell in the target column must match a regular expression. TBL-004 reads each table row and
reports a finding when a cell's value (after trimming) does not match `pattern`.

This enforces formatting conventions that an enumerated set cannot capture — an `ID` column that
must look like `REQ-123`, a `Date` column that must be ISO `YYYY-MM-DD`, a `Version` column that
must be semver. Where [TBL-003](TBL-003.md) checks membership in a fixed list, TBL-004 checks
structure.

If `section` is set, only tables under a heading whose text matches that section are checked.
Tables that do not declare the column are skipped.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `column` | `string` | **yes** | — | Header name of the column whose cells are constrained. |
| `pattern` | `string` | **yes** | — | Regular expression source; a cell is reported when its value does not match. |
| `flags` | `string` | no | — | Regex flags (e.g. `i`, `m`) applied when compiling `pattern`. |
| `section` | `string` | no | — | Only check tables under a heading matching this text; omit to check all tables. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "TBL-004", "options": { "column": "ID", "pattern": "^REQ-\\d+$" } }
  ]
}
```

### Passes

```md
| ID      | Owner |
| ------- | ----- |
| REQ-1   | Ann   |
| REQ-42  | Bea   |
```

### Fails

```md
| ID     | Owner |
| ------ | ----- |
| BUG-1  | Ann   |
| req-2  | Bea   |
```

→ TBL-004 reports each cell whose value does not match `^REQ-\d+$` — here both `BUG-1` and
`req-2` — anchored at their respective rows. Add `"flags": "i"` if the match should be
case-insensitive.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- Cell values are **trimmed** before matching, so surrounding whitespace does not affect the
  result.
- The pattern is a partial match unless you anchor it with `^`/`$`. Use anchors when the whole
  cell must conform, as in the example above.
- Because `pattern` is a JSON string, backslashes must be escaped (`\\d`, `\\.`).

# REF-004 — Cross-zone links are declared in the zone's Dependencies section

> Category **REF** (references) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

When your documentation is organized into **zones** — self-contained areas under a common
directory — REF-004 enforces that a link crossing from one zone into another is only allowed if
the source zone has *declared* that dependency. It reports a finding for any cross-zone link whose
target zone is not listed in the source zone's Dependencies section.

A zone is the first directory segment beneath `zonesDir`: a file at `zones/auth/page.md` belongs
to zone `auth`. Files directly under `zonesDir` (with no intermediate directory) have no zone and
are skipped.

To decide what a zone depends on, REF-004 collects the body of every Dependencies section
(default heading text `Dependencies`) across all files in the source zone, then records which
other zone names appear in that text. A link from the source zone to a *different* zone is
allowed only when the target zone was declared this way; links within the same zone are always
allowed.

This is a **local-only** check: it reads the corpus to learn zone membership and declarations and
does not perform network access.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `zonesDir` | `string` (non-empty) | **yes** | — | Directory whose first-level subdirectories define zones (e.g. `zones`). |
| `dependencySection` | `string` | no | `"Dependencies"` | Heading text whose section body lists the zones this zone may depend on. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "REF-004", "options": { "zonesDir": "zones" } }
  ]
}
```

### Passes

`zones/auth/page.md`:

```md
## Dependencies

- billing

## Body

See [billing](../billing/x.md).
```

→ Zone `auth` declares `billing`, so the cross-zone link to `billing` is allowed and REF-004
reports nothing.

### Fails

`zones/auth/page.md`:

```md
## Dependencies

- billing

## Body

See [billing](../billing/x.md) and [payments](../payments/y.md).
```

→ REF-004 reports that the **cross-zone link to zone `payments` is not declared in the
`Dependencies` section of zone `auth`**. The link into `billing` still passes because it is
declared.

### Custom dependency heading

```jsonc
{ "rule": "REF-004", "options": { "zonesDir": "zones", "dependencySection": "Depends On" } }
```

Declarations are read from `## Depends On` sections instead of `## Dependencies`.

## Notes

- **Scope:** `document` — evaluated per file, but the rule reads the corpus to learn zone
  membership and to gather Dependencies declarations across the whole source zone.
- A zone name is matched as a whole token in the Dependencies section body, so a substring of a
  longer word does not count as a declaration.
- Only local-file links are considered; anchors, external schemes, and same-zone links are
  ignored.

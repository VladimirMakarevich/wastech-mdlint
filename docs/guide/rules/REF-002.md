# REF-002 ‚Äî Link anchors match a heading slug

> Category **REF** (references) ¬∑ Scope **document** ¬∑ Default severity **error** ¬∑ Fixable **no**
> ¬∑ [Rules index](README.md) ¬∑ [Configuration](../configuration.md)

## What it checks

Every link anchor in scope must match a heading in the file it points at. REF-002 reads each
link's `#anchor` fragment and reports a finding when no heading in the target file produces that
slug.

Heading slugs are computed with `github-slugger`, so `## Getting Started` produces the slug
`getting-started`. The anchor in a link is compared against the slugs of the target file's
headings.

Two anchor forms are handled:

- **Same-file anchors** (`[jump](#getting-started)`) are validated against the linking document's
  own heading slugs.
- **Cross-file anchors** (`[jump](guide.md#getting-started)`) are validated against the resolved
  target file's heading slugs.

If a cross-file target file cannot be resolved, REF-002 skips it ‚Äî a missing target file is
[REF-001](REF-001.md)'s concern, not this rule's. Links without an anchor are ignored, and only
same-file and local-file links participate; other schemes are skipped. This is a **local-only**
check with no network access.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `siteRouter` | `object` | no | inherits `settings.siteRouter` | Overrides the site router used to resolve cross-file targets for this rule instance. See [Site router](#site-router). |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | ‚Äî | Glob(s) removing files from this rule instance. |

The `siteRouter` object accepts `preset`, `contentDir`, and `defaultLocale` (all optional
strings).

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "REF-002" }
  ]
}
```

### Passes

```md
## Intro

Jump to the [intro](#intro) or the [overview](b.md#overview).
```

with `b.md`:

```md
## Overview
```

‚Üí Both anchors match a heading slug, so REF-002 reports nothing.

### Fails

```md
## Intro

Jump to the [missing section](#nope) or a [bad cross-link](b.md#missing).
```

‚Üí REF-002 reports that anchor **`#nope` does not match any heading** in the current file, and
that anchor **`#missing` does not match any heading** in `b.md`.

## Site router

`settings.siteRouter` is **inherited by reference rules** and drives how root-relative cross-file
targets are resolved to source files before their headings are read; a per-rule `siteRouter`
option overrides the inherited value for that instance. See [REF-001](REF-001.md#site-router)
for the mapping details and locale fallback.

## Notes

- **Scope:** `document` ‚Äî evaluated per file, but cross-file anchors consult the resolved
  target's headings from the corpus.
- Slug comparison uses `github-slugger`; it does not accept arbitrary casing or spacing variants.
- Unresolvable target files are skipped so REF-002 and [REF-001](REF-001.md) do not both report
  the same broken link.

# GRP-002 ‚Äî Documents have an incoming reference

> Category **GRP** (graph integrity) ¬∑ Scope **project** ¬∑ Default severity **warning** ¬∑ Fixable **no**
> ¬∑ [Rules index](README.md) ¬∑ [Configuration](../configuration.md)

## What it checks

Every document in the repository must have at least one **incoming reference** ‚Äî some other
document that links to, imports, or otherwise points at it ‚Äî unless it is declared an **entry
point**. GRP-002 reads the shared [ContextGraph](../context-graph.md) and reports each node whose
in-degree is zero, catching orphaned files that nothing else reaches.

In-degree counts every kind of reference edge into the document: plain links, in-page anchor
links (a link to `detail.md#section` counts as reaching `detail.md`), image references, and eager
`@import` directives. When [`settings.idRef`](../configuration.md) is configured, `id-ref` edges
count too ‚Äî a document referenced only by its ID from elsewhere is *not* an orphan.

Because this rule reasons about relationships *between* files, its scope is `project`: it runs
once over the whole corpus. Findings are anchored at the top of the orphaned file.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `entryPoints` | `string[]` | no | ‚Äî | Glob(s) for documents allowed to have no incoming reference (roots like `README.md`, `index.md`). |
| `siteRouter` | `{ preset?, contentDir?, defaultLocale? }` | no | ‚Äî | Site-router hints accepted for forward-compatibility with route-aware analysis. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which orphaned files this rule instance reports. |
| `exclude` | `string[]` | no | ‚Äî | Glob(s) removing files from this rule instance's reporting. |

> **Note:** the shared ContextGraph always covers the whole corpus, so `files`/`exclude` narrow
> *which orphans get reported*, not which documents the graph sees. A file excluded from reporting
> still contributes its outgoing edges to other documents' in-degree.

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "GRP-002", "options": { "entryPoints": ["index.md"] } }
  ]
}
```

### Passes

```md
<!-- index.md -->
See [a](a.md).

<!-- a.md -->
# A
```

`index.md` is a declared entry point (no incoming reference required); `a.md` is reached from it.

### Fails

```md
<!-- index.md -->
See [a](a.md).

<!-- a.md -->
# A

<!-- orphan.md -->
# Orphan
```

‚Üí GRP-002 reports **`orphan.md has no incoming references; link it from another document or mark
it an entry point.`** ‚Äî nothing points at `orphan.md`, and it is not an entry point.

### Anchor links still count

```md
<!-- index.md -->
See [a](a.md).

<!-- a.md -->
See [the detail](detail.md#detail-heading).

<!-- detail.md -->
## Detail Heading
```

‚Üí No finding: the anchor link from `a.md` reaches `detail.md`, giving it an incoming reference.

## Notes

- **Scope:** `project` ‚Äî evaluated once over the whole corpus, not per file.
- **Default severity is `warning`**, not `error` ‚Äî orphaned files are often intentional (drafts,
  templates), so declare them as `entryPoints` to silence the finding.
- **Edges considered:** links, anchors, images, `@import`, and `id-ref` (the last only when
  `settings.idRef` is set). See [Context graph](../context-graph.md).

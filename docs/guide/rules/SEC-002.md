# SEC-002 — Sections in required order

> Category **SEC** (sections) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

The sections listed in `order` must appear in that relative order within a file. SEC-002 walks the
file's headings and reports the first section that appears *before* a section that should precede
it.

Only sections that are actually present are ordered — presence is [SEC-001](SEC-001.md)'s job, so
SEC-002 silently skips any name in `order` that the file does not contain. This means the two rules
compose: SEC-001 guarantees the sections exist, SEC-002 constrains their sequence.

By default every heading depth participates. Set `level` to compare only headings at one depth
(e.g. `2` considers only `##` headings). Set `section` to restrict the check to the run of headings
nested under a given parent section, up to the next heading of the same or higher level.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `order` | `string[]` (≥1) | **yes** | — | Section names in the sequence they must appear in. |
| `level` | `integer` (≥1) | no | all depths | Only consider headings at this depth (e.g. `2` = `##`). |
| `section` | `string` | no | whole file | Restrict the check to headings nested under this parent section. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "SEC-002", "options": { "order": ["Overview", "Usage"] } }
  ]
}
```

### Passes

```md
## Overview

## Usage
```

### Fails

```md
## Usage

## Overview
```

→ SEC-002 reports that section **`Usage`** appears before **`Overview`** but should come after it,
anchored at the `## Usage` heading.

### Restricted to one depth

```jsonc
{ "rule": "SEC-002", "options": { "order": ["Overview", "Usage"], "level": 2 } }
```

Only `##` headings are ordered; `###` subsections and the `#` title do not affect the check.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- **Not fixable:** reordering sections is a judgment call (bodies must move with their headings), so
  SEC-002 reports rather than rewrites.
- Only the **first** inversion is reported per file; fix it and re-run to surface any later one.
- Names absent from the file are ignored here — pair SEC-002 with [SEC-001](SEC-001.md) to also
  require the sections exist.
- Section comparison is on exact heading text; it does not normalize case or punctuation.

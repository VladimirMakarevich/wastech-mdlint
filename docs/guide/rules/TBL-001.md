# TBL-001 — Required table columns

> Category **TBL** (tables) · Scope **document** · Default severity **error** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Every Markdown table in scope must declare a set of required **header columns**. TBL-001 reads
each table's header row and reports a finding when a required column name is missing from it.

This catches tables that drift away from an agreed schema — e.g. a requirements table that must
always carry `ID`, `Owner`, and `Status` columns. It checks that the columns *exist* (by header
name); use [TBL-002](TBL-002.md) to require their cells be non-empty, and
[TBL-003](TBL-003.md)/[TBL-004](TBL-004.md) to constrain cell values.

Column matching is by trimmed header text. If `section` is set, only tables under a heading whose
text matches that section are checked; otherwise every table in the file is checked.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `requiredColumns` | `string[]` (≥1) | **yes** | — | Header names every in-scope table must declare. |
| `section` | `string` | no | — | Only check tables under a heading matching this text; omit to check all tables. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "TBL-001", "options": { "requiredColumns": ["ID", "Owner", "Status"] } }
  ]
}
```

### Passes

```md
| ID    | Owner | Status |
| ----- | ----- | ------ |
| REQ-1 | Ann   | done   |
```

### Fails

```md
| ID    | Status |
| ----- | ------ |
| REQ-1 | done   |
```

→ TBL-001 reports that the table is missing the required column **`Owner`**, anchored at the
table's header line.

### Scoped to one section

```jsonc
{ "rule": "TBL-001", "options": { "requiredColumns": ["ID"], "section": "Requirements" } }
```

Only tables beneath a `Requirements` heading are checked; a table under `Changelog` is ignored.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- **`files`/`exclude`** let you register the same rule multiple times with different column sets
  for different document families (e.g. one schema for `docs/requirements/**`, another elsewhere).
- Header comparison is on trimmed text; it does not normalize case or punctuation.

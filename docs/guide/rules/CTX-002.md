# CTX-002 — All checklist items are checked

> Category **CTX** (content) · Scope **document** · Default severity **warning** · Fixable **no**
> · [Rules index](README.md) · [Configuration](../configuration.md)

## What it checks

Every Markdown task-list item (`- [ ]` / `- [x]`) in scope must be **checked**. CTX-002 reports a
finding for each item left unchecked.

This is the rule for **checklist completeness** — there is no separate `CHK` category. Use it to
gate documents that carry a definition-of-done or acceptance checklist, so a document with an
open `- [ ]` item is surfaced.

If `section` is set, only items under a heading whose text matches it are considered; otherwise
every task-list item in the file is checked.

## Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `section` | `string` | no | — | Only check items under a heading matching this text; omit to check all items. |
| `files` | `string[]` | no | all included | Glob(s) narrowing which files this rule instance applies to. |
| `exclude` | `string[]` | no | — | Glob(s) removing files from this rule instance. |

## Example

### Config

```jsonc
{
  "rules": [
    { "rule": "CTX-002" }
  ]
}
```

### Passes

```md
## Tasks

- [x] Write the parser
- [x] Wire up the CLI
```

### Fails

```md
## Tasks

- [x] Write the parser
- [ ] Wire up the CLI
```

→ CTX-002 reports that the checklist item **`Wire up the CLI`** is not checked, anchored at that
item's line. Each unchecked item produces its own finding.

### Scoped to one section

```jsonc
{ "rule": "CTX-002", "options": { "section": "Tasks" } }
```

Only task-list items beneath a `Tasks` heading are checked; unchecked items elsewhere (e.g. a
`Future ideas` list) are ignored.

## Notes

- **Scope:** `document` — evaluated per file, independent of other files.
- One finding is emitted per unchecked item, so the count reflects remaining work.
- **`files`/`exclude`** let you apply completeness enforcement only to the document families that
  carry actionable checklists.
